1. Introdução

O Algoritmo de Euclides é um dos algoritmos mais antigos e elegantes da matemática, desenvolvido pelo matemático grego Euclides por volta de 300 a.C. Este algoritmo resolve o problema fundamental de encontrar o Máximo Divisor Comum (MDC) entre dois números inteiros de forma extremamente eficiente.

Importância Histórica

Descrito originalmente no Livro VII dos "Elementos" de Euclides, este algoritmo representa um dos primeiros exemplos documentados de um algoritmo sistemático. Sua elegância reside na simplicidade da ideia: o MDC de dois números não muda quando substituímos o maior número pela diferença entre os dois números.

Aplicações Práticas Modernas

  • Criptografia RSA: Fundamental para gerar chaves criptográficas e verificar se dois números são coprimos
  • Simplificação de frações: Reduzir frações à forma irredutível
  • Algoritmos de teoria dos números: Base para outros algoritmos como o Algoritmo Extendido de Euclides
  • Computação gráfica: Geração de padrões e tessellações
  • Análise de algoritmos: Exemplo clássico de análise de complexidade logarítmica
Definição do Problema: Dados dois números inteiros positivos a e b, encontrar o maior número inteiro que divide ambos sem deixar resto, conhecido como MDC(a,b) ou gcd(a,b).

2. Descrição do Algoritmo

Implementação em JavaScript

// Versão Recursiva do Algoritmo de Euclides
function mdcRecursivo(a, b) {
    // Caso base: quando b = 0, o MDC é a
    if (b === 0) {
        return a;
    }
    // Chamada recursiva: MDC(a, b) = MDC(b, a % b)
    return mdcRecursivo(b, a % b);
}

// Versão Iterativa (otimizada)
function mdcIterativo(a, b) {
    // Garante que trabalhamos com valores absolutos
    a = Math.abs(a);
    b = Math.abs(b);

    // Loop enquanto b não for zero
    while (b !== 0) {
        let temp = b;
        b = a % b;  // Novo b é o resto da divisão
        a = temp;   // Novo a é o antigo b
    }
    return a;
}

// Versão com rastreamento de passos para animação
function mdcComPassos(a, b) {
    let passos = [];
    a = Math.abs(a);
    b = Math.abs(b);

    passos.push(`Iniciando: MDC(${a}, ${b})`);

    while (b !== 0) {
        let quociente = Math.floor(a / b);
        let resto = a % b;

        passos.push(`${a} = ${b} × ${quociente} + ${resto}`);
        passos.push(`MDC(${a}, ${b}) = MDC(${b}, ${resto})`);

        a = b;
        b = resto;
    }

    passos.push(`Resultado: MDC = ${a}`);
    return { resultado: a, passos: passos };
}

3. Análise Matemática de Complexidade

Análise de Recorrência

Para analisar a complexidade do Algoritmo de Euclides, consideremos a relação de recorrência:

T(a, b) = T(b, a mod b) + O(1)

O fator determinante é quantas iterações são necessárias até que b = 0.

Teorema de Lamé (1844)

O número de passos no Algoritmo de Euclides para calcular MDC(a, b) onde a > b > 0 é no máximo 5 vezes o número de dígitos decimais de b.

Se Fn é o n-ésimo número de Fibonacci, então:
Número de passos ≤ logφ(√5 × b)
onde φ = (1 + √5)/2 ≈ 1.618 (razão áurea)

Demonstração da Complexidade O(log n)

Caso Pior: Números consecutivos de Fibonacci
Fn+1 e Fn requerem exatamente n-1 passos

Análise Detalhada:

  1. Redução a cada passo: Em cada iteração, os argumentos diminuem significativamente
  2. Propriedade fundamental: Se a > b, então a mod b < a/2
  3. Consequência: A cada dois passos, pelo menos um dos argumentos é reduzido pela metade
  4. Resultado: O(log min(a, b)) passos no total

Complexidades por Caso

Caso Complexidade de Tempo Complexidade de Espaço Exemplo
Melhor Caso O(1) O(1) MDC(15, 5) = 5 (um passo)
Caso Médio O(log n) O(1) iterativo / O(log n) recursivo Maioria dos casos práticos
Pior Caso O(log n) O(1) iterativo / O(log n) recursivo Números de Fibonacci consecutivos

Notações Assintóticas

Big-O (Limite Superior): T(n) = O(log n)
Ω (Limite Inferior): T(n) = Ω(1)
Θ (Limite Justo): T(n) = Θ(log n) para o caso médio

5. Discussão e Comparação

Validação da Análise Teórica

Os resultados empíricos confirmam consistentemente a análise teórica de complexidade O(log n). A correlação entre teoria e prática é especialmente evidente quando analisamos:

  • Crescimento logarítmico: O tempo de execução cresce muito lentamente mesmo para números grandes
  • Caso pior confirmado: Números de Fibonacci consecutivos realmente apresentam o maior número de iterações
  • Eficiência prática: O algoritmo mantém alta performance mesmo para números de milhões de dígitos

Possíveis Divergências

Fatores que podem causar divergências:
  • Overhead de função em JavaScript para números pequenos
  • Otimizações do motor JavaScript (V8) podem mascarar diferenças
  • Garbage collection pode introduzir ruído nas medições
  • Precisão limitada de performance.now() para operações muito rápidas

Fatores que Influenciam o Desempenho

Implementação Recursiva vs Iterativa:

  • Recursiva: Mais elegante, mas usa O(log n) espaço na pilha
  • Iterativa: O(1) espaço, geralmente mais rápida na prática

Características dos Números de Entrada:

  • Razão entre os números: Números com razão próxima à razão áurea são mais lentos
  • Números coprimos: Tendem a ter mais iterações
  • Múltiplos óbvios: Terminam rapidamente

Limitações Observadas

  • Precisão numérica: JavaScript possui limitações para inteiros muito grandes (> 2^53)
  • Overhead de medição: Para números pequenos, o tempo de medição pode ser maior que o de execução
  • Variabilidade do sistema: Outros processos podem afetar as medições

Cenários de Uso Recomendados

Use o Algoritmo de Euclides quando:
  • Precisar calcular MDC de forma eficiente
  • Implementar algoritmos criptográficos
  • Trabalhar com frações (simplificação)
  • Resolver problemas de teoria dos números
  • Necessitar de um algoritmo com garantia de complexidade logarítmica

Comparação com Outros Métodos

Método Complexidade Vantagens Desvantagens
Algoritmo de Euclides O(log n) Rápido, elegante, teoricamente sólido Requer divisão
Fatoração Prima O(√n) Fornece fatores completos Muito lento para números grandes
Força Bruta O(min(a,b)) Simples de implementar Ineficiente para números grandes
